核心概念映射 (GitHub -> GitHabit)

- User Profile (用户主页) -> 仪表盘 (Dashboard)：展示整体打卡情况的“绿格子”热力图（Contribution Graph），以及置顶的习惯（Pinned Repositories）。
- Repository (代码仓库) -> 习惯/计划 (Habit)：创建一个 Habit 就像建一个 Repo。可以包含名称、描述、语言（可映射为习惯分类，如“健康”、“学习”等）。
- Commit (提交) -> 打卡 (Check-in)：完成一次习惯就是进行一次 Commit。
Commit Message：打卡时的备注/心情。
Commit 频率：决定了绿格子的颜色深浅（浅绿到深绿）。
- Activity/Commit History (提交历史) -> 打卡时间线：以时间轴列表形式展示某项习惯的历次打卡记录。
- Issues (议题) -> 待办事项/子任务 (Todos)：（可选）如果某个习惯比较复杂，可以像提 Issue 一样拆解子任务。
